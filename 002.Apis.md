# âœ… **4. IActionResult / ActionResult**

**The MOST flexible and enterprise-level best practice.**

### Using `IActionResult`

```csharp
public IActionResult Get(int id)
{
    if (id <= 0) return BadRequest();

    return Ok(new { Message = "Success" });
}
```

### Using `ActionResult<T>`

```csharp
public ActionResult<UserDto> GetUser(int id)
{
    if (id <= 0) return NotFound();

    return new UserDto { Id = 1, Name = "Test" };
}
```

# ðŸ”¥ **5. Built-in Http Response types (ObjectResult family)**

| Return                            | Meaning        |
| --------------------------------- | -------------- |
| `Ok(object)`                      | 200 OK         |
| `Created()` / `CreatedAtAction()` | 201 Created    |
| `Accepted()`                      | 202 Accepted   |
| `NoContent()`                     | 204 No Content |
| `BadRequest()`                    | 400            |
| `Unauthorized()`                  | 401            |
| `Forbidden()`                     | 403            |
| `NotFound()`                      | 404            |
| `Conflict()`                      | 409            |
| `StatusCode(500)`                 | Custom status  |

These all implement `IActionResult`.

# âœ… **7. HttpResponse / Raw Response**

Manually writing to the response body:

```csharp
public async Task Get()
{
    Response.ContentType = "text/plain";
    await Response.WriteAsync("Hello");
}
```

Use only when needed (e.g., streaming, file output).
